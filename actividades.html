A.1 Encuestas a los usuarios (0.5 sem)Cómo se hace: Diseñar un cuestionario sobre tiempos de espera, satisfacción y mejoras deseadas. Aplicarlo presencialmente o por formularios digitales (Google Forms, Typeform). Analizar respuestas para identificar patrones. Recursos: Plataforma de encuestas, dispositivos móviles o impresos, acceso a usuarios. A.2 Estudiar métodos actuales (0.5 sem) Cómo se hace: Revisar sistemas de gestión de filas en otras empresas, estudiar software similar (turnos digitales, citas previas). Analizar documentación técnica y realizar entrevistas con empleados o clientes. Recursos: Documentación de software, informes de uso, entrevistas, Internet. A.3 Definir funcionalidades específicas (0.5 sem) Cómo se hace: Con base en encuestas y estudio de métodos, definir qué funciones tendrá el sistema (turnos virtuales, notificaciones, estimación de espera). Crear una lista priorizada y validar con stakeholders. Recursos: Reuniones con el equipo, software de gestión de proyectos (Trello, Notion), documentos de especificaciones. A.4 Documentar requerimientos Cómo se hace: Redactar un documento con requisitos funcionales y no funcionales, diagramas de flujo y casos de uso. Validar con el equipo antes de finalizar. Recursos: Procesador de texto (Google Docs, Word), herramientas de diagramación (Draw.io, Lucidchart), retroalimentación de stakeholders.
